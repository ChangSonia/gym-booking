# 從這裡開始 — Claude Code 使用教學

**你沒有程式背景，這份文件就是為你寫的。**

---

## Claude Code 是什麼

一個裝在你電腦裡的 Claude，但它**看得到你的檔案、能跑指令、能自己修錯**。

**現在（在網頁版聊天）：**
```
你：「候補中改成等待中」
Claude：改一行 → 貼給你
你：複製 → 貼上 → 上傳 GitHub → 等部署 → 看畫面
你：「壞了」
Claude：「把錯誤訊息貼給我」
```

**Claude Code：**
```
你：「候補中改成等待中」
它：自己找到 3 個檔案裡的 5 個地方 → 改掉 → 跑測試 → 部署
```

**你不用記語法，用中文講就好。**

---

## 第一次使用

### 1. 開啟 Claude 桌面版 → Code 分頁

你已經有桌面版了。左側或上方會有 **Code** 的分頁，點它。

### 2. 建一個資料夾

在你的電腦上建一個資料夾，例如：

```
C:\Users\你的名字\Documents\gym-booking
```

**把這包檔案全部放進去。**

### 3. 在 Claude Code 開啟這個資料夾

Code 分頁會問你要開哪個資料夾，選剛剛那個。

### 4. 第一句話這樣說

```
請先讀 CLAUDE.md 和 docs/DECISIONS.md，
然後告訴我你理解的專案現況，以及你建議的第一步。
```

**它會自己讀完所有文件**，然後跟你確認。這比你重講一次快得多。

## 你會用到的話術

Claude Code 聽得懂中文，但講清楚一點效果比較好。

### 好的講法

| 你想做的事 | 這樣說 |
|---|---|
| 建專案 | `照 CLAUDE.md 的技術棧，建一個 Next.js 專案` |
| 接資料庫 | `我的 Supabase 已經建好了，幫我接上去` |
| 改文案 | `把候補中改成等待中，所有地方都要改` |
| 修 bug | `報名之後畫面沒更新，幫我找原因` |
| 部署 | `幫我部署到 Vercel` |
| 看懂程式 | `這個檔案在做什麼？用白話解釋` |

### 不好的講法

| 不好 | 為什麼 |
|---|---|
| `改一下` | 改什麼？ |
| `把那個弄好` | 哪個？ |
| `照之前那樣` | 它不知道「之前」 |

### 遇到問題時

**直接把錯誤訊息整段貼給它。** 不用翻譯、不用整理，原封不動貼上去就好。

```
我跑起來看到這個：
Error: Cannot find module 'next'
    at Function.Module._resolveFilename...
```

它會自己看懂並修。

## 重要習慣（會救你一命）

### 1. 每做完一件事就 commit

```
幫我 commit，訊息寫「加入候補遞補功能」
```

**commit = 存檔點。** 改壞了可以回到上一個 commit。

不 commit 就一直改，改壞了會很想哭。

### 2. 讓它先講計畫再動手

```
先跟我說你打算怎麼做，我確認之後再開始
```

避免它跑去改一堆你沒預期的東西。

### 3. 產品決策要問你，不要它自己決定

`CLAUDE.md` 裡已經寫了「改動前先問」，但你也可以提醒：

```
這是產品決策還是技術實作？如果是產品決策，先問我
```

**例子**：「候補要不要跳號」是老闆的營運判斷，不是技術問題。

### 4. 不要一次要它做十件事

```
❌ 幫我做完整個系統
✅ 先幫我把資料庫接起來，能讀到課表就好
```

小步前進，每步都確認能跑。

## 建議的開發順序

**不要跳步。** 每一步都要能跑起來、看得到東西，再做下一步。

```
第 1 步  Next.js 專案跑起來，瀏覽器看到 Hello World
第 2 步  接 Supabase，能讀到課表資料
第 3 步  LIFF 登入，能拿到 LINE 身分
第 4 步  課表頁面（唯讀，先看得到就好）
第 5 步  報名 / 取消（核心功能）
第 6 步  候補 + 遞補
第 7 步  教練後台
第 8 步  推播通知
第 9 步  部署到 Vercel
第 10 步 排程器（每天排新課、歸檔舊課）
```

**第 1-3 步是地基**，卡住很正常，慢慢來。過了第 3 步之後會越來越快。

## Supabase 怎麼設定

1. 開 https://supabase.com → **New project**
2. 名稱 `gym-booking`，密碼**存起來**（很重要，會用到）
3. Region 選 **Northeast Asia (Tokyo)** ← 離台灣最近
4. 等 2 分鐘建好
5. 左側 **SQL Editor** → **New query**
6. 把 `supabase/schema.sql` **整個貼進去** → **Run**
7. 左側 **Table Editor** → 應該看得到 `courses`、`sessions` 等表，裡面有你們的課表

### 拿金鑰

左側 **Settings** → **API**：

| 名稱 | 貼到 .env 的哪裡 |
|---|---|
| Project URL | `SUPABASE_URL` |
| `anon public` | `SUPABASE_ANON_KEY` |
| `service_role` | `SUPABASE_SERVICE_KEY` ⚠️ **絕對不能外流** |

> **service_role key 等於資料庫的萬能鑰匙。** 只能放在後端，絕對不能出現在前端程式碼裡。

### 把自己設成管理員

等你用 LINE 登入過一次之後：

1. Table Editor → `users` 表
2. 找到你自己那一列
3. `is_admin` 和 `is_coach` 點成 `true`
4. 存檔

**這是雞生蛋問題的解法**：第一個管理員必須手動設。

## 之後改東西的三種難度

| 類型 | 例子 | 誰能改 |
|---|---|---|
| **資料** | 加開課、改名額、換教練、停課 | 老闆自己在 app 裡點 |
| **設定** | 開放時間改 10:00、上限改 6 人 | 你在 Supabase 點 `settings` 表 |
| **程式** | 文案、版面、新功能 | Claude Code 改 → 自動部署 |

**這就是為什麼課表要放資料庫，不能寫在程式裡。**
老闆說「泰拳改 18 人」，如果要你改 code 重新部署，你會被煩死。

## 卡住的時候

1. **把錯誤訊息整段貼給 Claude Code**
2. 還是不行 → `這個問題我卡住了，請一步一步排查，先告訴我你要檢查什麼`
3. 真的不行 → 回來網頁版問我，把 Claude Code 的對話貼過來

**你會遇到很多錯誤，那是正常的。** 專業工程師每天也在看錯誤訊息。

差別只在於他們知道「錯誤訊息是線索，不是責備」。貼給 Claude Code，它會處理。
